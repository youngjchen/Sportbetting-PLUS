# 層3 判讀表製作程序（辯論協議）— v1，執行前凍結

> 本文件定義層3（整詩＋典故）判讀表的**唯一**製作程序。程序凍結後執行一次，不重跑、不挑 run。
> 產出之 60 行判讀表雜湊將與本文件雜湊一併列入凍結清單。
> 執行平台：Claude Code Workflow（多 agent），2026-07-06。

## 0. 原則

- 解讀者全程**盲於一切比賽資訊**：材料只有籤文/斷語/典故；所問之事以抽象「一注」表述，解讀者不知領域、不知日期、不知對象。
- 判讀表按籤預製、與比賽無關 → 結構性堵死「向命中率優化」與「塞真實資訊」。
- 使用者終審只能「維持」或「降級為棄」，不可翻向。

## 1. 三位解讀者（互盲，各自獨立呼叫）

**共同前導（逐字）**：

> 你是一位傳統六十甲子籤解籤人。以下是一首籤的材料。問卜者所問之事固定為：「問卜者欲下一注（單一事項之注：成則得利，敗則折損），此籤對此注之吉凶如何？」
> 規則：
> - 只能依據所給材料判讀，禁止引入任何外部資訊或猜測問卜者背景。
> - 判決只能是「吉」（此注可為）、「凶」（此注不可為）、「棄」（籤意不明、中平、或無法對此注表態）三者之一。
> - 必須附引文：至少一句詩文原句或典故情節作為判決依據；引不出依據就判「棄」。

**解讀者Ａ（文本派，model=sonnet）**：追加指示「你只依詩文字面與斷語（求財、凡事）判讀，不使用典故。」材料＝詩文＋五行＋官方求財斷語＋官方凡事斷語。

**解讀者Ｂ（典故派，model=opus）**：追加指示「你以典故故事的結構為主要依據（主角最終成敗、先困後成、先成後敗、有無逆轉），詩文為輔。」材料＝詩文＋典故標題＋故事情節。

**解讀者Ｃ（整體派，model=主會話模型）**：追加指示「你綜合詩文、斷語、典故整體判讀，如廟中解籤人。」材料＝全部。

**輸出 schema（強制）**：`{verdict: "吉"|"凶"|"棄", citation: string, reason: string(≤120字)}`

## 2. 批評者（每籤一位，model=主會話模型）

**prompt（逐字）**：

> 你是解籤審查人。三位解讀者對同一首籤給出了判決（附材料與三份判決）。你的任務是嘗試推翻「多數判決」：檢查其引文是否真實存在於材料中、是否斷章取義、材料中是否有被忽略的明確反向詩句或情節。
> sustained=true 的標準（從嚴）：多數判決的引文不存在或被斷章取義，或材料中有明確反向證據。僅僅「另一種解讀也說得通」不足以推翻。

**輸出 schema**：`{sustained: boolean, reason: string(≤120字)}`

## 3. 裁決規則（程式執行，無裁量）

1. 三票中某判決獲**絕對多數（≥2票）** → 採納該判決；2-1 時記錄異議（dissent）。
2. 無絕對多數（吉/凶/棄各一）→ **棄**。
3. 多數判決為「棄」→ 棄。
4. 批評者 sustained=true → **棄**（記錄理由）。多數判決為「棄」時批評者步驟略過（推翻亦不改變結果，規則 3 已定棄）。
5. 使用者終審：對每行只能「維持」或「降級為棄」。終審完成 → 表定稿 → 雜湊入凍結清單。

## 4. 附錄：轉錄工序（前置，非辯論之一部分）

每籤一位轉錄 agent（model=主會話模型，具視覺）：讀官方掃描圖，逐字轉錄 求財格、凡事格、典故標題、故事欄大意（2-3句）、詩文覈對。指示明載「以圖為準；chance 參考值僅用於定位欄位，不同時以圖為準並標記 discrepancy」。轉錄結果＝層1/層2 序位表與辯論材料（求財/凡事）的權威來源；與 chance 之出入全數留檔。

## 5. v1.1 修訂（2026-07-06，於任何辯論票產生**之前**定案——兩次額度中斷檢討後）

1. **材料規則**：解讀者之【典故／情節】一律改用**官方圖轉錄**（storyTitles＋storyGist）；chance 網站之故事全文**排除**——轉錄已證實其典故存在張冠李戴（第2/4/5籤圖上標題與 chance 所附故事不符），錯誤歸屬會污染典故派判讀。
2. **輸出瘦身**：判決 reason ≤80 字（原 ≤120）；citation ≤40 字。判準語義不變。
3. **執行工程**：材料全數內嵌 prompt（禁共用大檔讀取）；分塊執行＋磁碟檢查點；連續失敗熔斷。凍結的是 §1–§3 之判讀語義與裁決規則，執行工程屬非語義層，得因額度紀律調整。
4. 三解讀者×模型配置不變（A=sonnet／B=opus／C=主模型）。

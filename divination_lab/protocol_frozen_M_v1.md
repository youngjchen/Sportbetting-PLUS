## 1. 可證偽主張（兩個獨立的 confirmatory 實驗）

### 實驗 M（梅花易數 × 歷史回測）
- **If**：以梅花易數時間起卦法（卦 = 原典演算法(比賽預定開打時刻)，完全確定性、無人參與）經鎖定映射產生大小分二元預測；
- **Then**：對 2019 至今全部合格場次，命中率顯著異於 50%（雙尾）；
- **Because**（受測假說）：占卜系統聲稱時刻本身攜帶事件走向的資訊。

### 實驗 L（六爻 × 前瞻）
- **If**：系統排程在賽前固定時刻，以物理真隨機源自動模擬三錢六擲起卦，經鎖定映射產生大小分二元預測；
- **Then**：前瞻累積的合格場次中，命中率顯著異於 50%（雙尾）；
- **Because**（受測假說）：占卜系統聲稱起卦瞬間的隨機過程攜帶未來事件的資訊。

**範圍聲明（雙向有效）**：兩實驗檢驗的都是「本協議鎖定的特定映射」。null 不證偽「占卜整體」或「真人解卦實踐」；正結果也不支持「占卜為真」，只觸發第 10 節的複製程序。

## 3. 主檢定與虛無值

- **主檢定**：占卜臂命中數 vs H₀: p = 0.5，**精確二項檢定，雙尾，α = 0.05**。
- **虛無值 0.5 的成立前提（分析前認證，兩項都要過）**：
  1. **映射端**：實驗 L 以 10⁶ 次模擬隨機卦跑過完整映射，押「大」比例須在 50%±0.5%；實驗 M 以全部合格場次的實際起卦時刻跑映射，記錄押「大」比例 p̂側。
  2. **結果端**：合格樣本的實際 Over 率 P̂over（預期 49–51%）。
  - 調整後虛無值 p₀ = p̂側·P̂over + (1−p̂側)(1−P̂over)。若 |p₀ − 0.5| > 0.5pp，主檢定改用 p₀；再不放心即降級為與對照臂的 McNemar 配對檢定。
- **認證結果（2026-07-03，全程結果盲）**：梅花臂以 17,938 個實際賽程時刻實測——比和僅 141 卦（0.79%）→ 依 §6 預定規則**排除**；排除後押大 p̂側 = **54.68%**（逐年 53.1–57.0%，穩定），成因＝MLB 開賽時刻集中於少數時辰的結構效應，與比和無關。±0.5pp 判準未過 → 依本節預定機制生效：主檢定虛無值採 p₀（P̂over 於分析日以合格樣本實測；p₀−0.5=(2p̂側−1)(P̂over−0.5)≈0.094×(P̂over−0.5)，公式精確吸收）；**日期分層置換檢定升格為共同主要穩健檢**；McNemar 對照臂備援不變。引擎自測 18/18（觀梅占算例、8 個正月初一錨點、時辰邊界、餘0 規則）。
- **穩健性檢查（一律附報，不作決策）**：
  - 置換檢定：日期分層內打亂結果重抽 null 分布（免分布假設）。
  - 實驗 M 的「同卦簇」聚類穩健檢定：同一時辰起卦的比賽共享同一卦與同一押向，以卦簇為單位重估標準誤。
  - 前後時段方向一致性（描述性，不設顯著門檻——取代舊版「訓練/驗證同向」關卡；回測樣本另以 2019–2022 / 2023–今 時間切分附報兩段結果）。

## 4. 有效／無效標準（事先寫死，SESOI = ±5pp）

| 結果情形 | 唯一允許的結論句 |
|---|---|
| 顯著，且 95% CI 下界 > 50%（或 < 50%） | 「此映射存在未被排除的訊號 → 觸發第 10 節預註冊複製。」**不得**寫「占卜有效」。 |
| 不顯著，且等效檢定（TOST，α=0.05）通過 ±5pp | 「在本樣本下，此映射的效應被排除在 ±5pp 之外＝實務上無效。」 |
| 不顯著，等效檢定未通過 | 「無資訊結果，僅報告 CI。」**不得**寫「證偽」。 |

- 等效檢定樣本需求（真效應=0、聯合 80% power）：±5pp 需 **≈857 場**；±3pp 需 ≈2,380 場；±1.3pp 需 ≈12,700 場。
- 兩份結論模板（null 版／正版）隨協議凍結，屆時只准填空。

## 6. 起盤規範（自動化，人手不碰）

- **實驗 M**：卦 = f(該場**預定開打時刻**)，f = 通行本卷一〈年月日時起例〉演算法（考據已核實有明文）。已採納自定值（公開聲明）：時刻=台北時間、0 點換日（23:00–23:59 為當日子時）；農曆干支轉換沿用八字專案已驗證引擎；除 8 餘 0=坤（有明文）、除 6 餘 0=上爻（**自定**）；動爻含時（起例明文）；先天卦數乾1…坤8。
- **實驗 M 映射（官訟版＋通則補全，已核可）**：體克用→體方勝；用克體→用方勝；用生體→體方勝；體生用→用方勝；**比和→體方勝**（若對稱性模擬偏差 >0.5pp，改排除比和場並重算樣本量）。極性：大小分 體=「大」方；讓分/獨贏 體=主隊。v1 不納旺衰/互變/爻辭/剋應（聲明為最大簡化；旺衰版另列 exploratory 第二映射）。
- **實驗 L 映射（已核可）**：世=「大」方（大小分）/主隊（讓分獨贏）；勝負依《卜筮正宗》征戰章「觀世應之旺衰，以決兩家之勝負」做確定性計分（月建/日辰生扶權值於引擎規格凍結前定案並聲明自創）；同分→世應本爻五行相剋；再平→依極性判世方。
- **實驗 L**：排程於**賽前 60 分鐘**自動起卦：取 NIST Randomness Beacon 該分鐘 pulse（記錄 pulse index 與原始值，任何人可事後驗證）→ 無偏位元抽取 → 模擬三錢六擲（老陰 1/8、少陽 3/8、少陰 3/8、老陽 1/8）→ 裝卦 → 映射 → 立即寫入 log 並 git commit（commit 時間戳＝公證）。一場一卦，**永不重抽**。
- **延賽**：改期則原卦作廢留痕，於新開打時刻依同規則重新自動起卦。
- **對照臂**：seed = HMAC-SHA256(key=SHA256(凍結協議全文), msg=gamePk)，生成同格式卦、套同一映射，與占卜臂同時寫入。任何人可重算驗證。
- **手動問事頁面**：保留為互動功能，數據一律標記 non-experimental，不入統計。

## 7. 資料與排除規則（接觸結果前寫死）

- 盤口來源：玩運彩運彩盤**收盤線**（快照時刻與欄位於爬蟲規格中固定）。
- 唯一鍵：**gamePk**（雙重賽教訓）。結果以 MLB 官方 API 為準，結算判定以運彩官方規則為準。
- 排除（全部留痕記數，不得事後增刪）：無盤場次；大小分 push；讓分和局/卡分退款場（僅影響 exploratory）；**7 局制雙重賽（2020–21）**；腰斬/未完賽依運彩結算規則處理並標記。
- 賽事類型（以 MLB API gameType 判定）：**春訓（S）與明星賽（A）排除；confirmatory＝例行賽（R）；季後賽（F/D/L/W）另標為 exploratory 子集**。（2019 回補實測：3 月下旬含春訓場且多無盤線；明星賽已被隊名診斷自動攔截。）
- 實測註記（2026-07-03）：運彩大小線 16,955 條**全為 .5 線**、讓分線僅 ±1.5/±2.5——push/卡分為**結構性空集合**（規則保留備查）；gamePk 未匹配 172 場（0.96%，多為延賽/續完賽時刻位移）與 ambiguous 146 場全留痕排除；7 局賽 263 場已旗標。
- 2020 縮水賽季的 9 局比賽保留（收盤線已定價當年規則）。

## 8. 多重比較與 run 註冊表

- Confirmatory 家族 = {實驗 M×大小分, 實驗 L×大小分}，兩個獨立主張各 α=0.05，互不佔用。
- 其餘一切（讓分、獨贏、其他占卜法、任何子群）＝exploratory，以 BH-FDR 10% 標註後僅作描述。
- **全域 run 註冊表**：每一個啟動的實驗（含中止、含 pilot）都必須入最終報告；「六爻不行換塔羅」允許，但每次換都是新註冊、全數公開——不存在無聲放棄。

## 9. 監控、遮蔽與絆線

- 期中儀表板只顯示：資料完整度、起卦成功率、兩臂押向分布。**命中率對照在分析日前遮蔽。**
- 診斷絆線（觸發即凍結調查）：各臂「押大比例」偏離**各自認證值**（梅花臂 54.68%；對照臂 50%——均勻卦下結構對稱，待其生成器完成後另以 10⁶ 次模擬認證）超過 ±2pp；起卦時間戳晚於開打；beacon pulse 無法驗證；gamePk 重複或缺漏。（修訂註：原「兩臂互比 >5pp」不適用——確定性臂與均勻臂的押向分布本就不同，此為認證後修訂、非事後改規則。）

## 10. 正結果的後續（predefined escalation）

顯著 → 依序排除世俗解釋（結算錯誤、洩漏、時鐘、押向偏差）→ 通過才觸發**預註冊複製**（新賽季、同映射、凍結不動）→ 複製再過，才允許討論不尋常解釋。單一 run 的 p<0.05 依 Sellke–Berger 上界至多 BF≈2.5，本協議明文承認這一點。

## 12. 凍結程序（一切就緒，等使用者口令）

1. 協議＋定義文件複製入 repo（`Sportbetting-PLUS/divination_lab/`），與六支腳本同倉。
